01-Quais as principais funções da camada de rede?

As principais funções da camada de rede é a distribuição da informação pela rede, fazendo com que as mensagens sejam entregues

aos seus destinos de uma forma eficiente e segura, soponhamos que hajam duas pessoas conectadas a uma mesma rede LAN, essas pessoas

podem ter suas mensagens enviadas diretamente de um computador para o outro sem que haja a necessidade de uma rede de longa distância,

porém, se eles estiverem conectados em redes diferentes essa conexão não seria possível sem a existência de uma camada de rede.

02- Explique o que é roteamento em redes IPv4 e como os roteadores tomam as decisões de roteamento/encaminhamento

O roteamento é o processo de seleção de caminho em qualquer rede, as redes são compostas por muitos nós e caminhos ou links

que conectam esses nós.

A comunicação entre os nós de uma rede interconectada pode ocorrer por meio de muitos caminhos diferentes e o reteamento é quem

decide qual o melhor caminho a ser seguido usando algumas regras determinadas.

Para esse processo são seguidas algumas regras, sendo elas:

  • Construção de pacote com endereço de destino do "PC" para quem deseja enviar a mensagem e o coloca dentro de um frame ethernet com o endereço MAC de destino do router e envia pela ligação ethernet
    1. O router R1 remove a header e o trailer Ethernet
    2. Compara o endereço de destino do pacote com a tabela de roteamento e encontra a entrada para a sub-rede.
    3. De seguida encapsula o pacote IP com uma header e trailer HDLC encaminha a frame para o próximo salto (hop) que é o router R2 através da interface de saída Serial
  • O router R2 processa a frame e encaminha-a para o R3. O R2 repete o processo de recepção da frame, verifica através do campo FCS se ocorreram erros.
  • O router R3 processa a frame e encaminha para o PC2. Repete o processo de recepção da frame e verifica no FCS que não ocorreram erros

Como os endereços IPv4 são compostos e quantos bits eles contêm? Quais as duas representações (notações) possíveis dos endereços IPv4?

Os endereços IPv4, como 192.168.0.1 , são apenas representações decimais de quatro blocos binários. Cada bloco tem 8 bits e representa números de 0 a 255. Como os blocos são
grupos de 8 bits, cada bloco é conhecido como um octeto. Pelo fato de haver quatro blocos de 8 bits, cada endereço IPv4 é de 32 bits.

O que é uma máscara de sub-rede IPv4 e como ela é usada para dividir redes em sub-redes? Quais as duas representações (notações) possíveis das máscara de sub-rede IPv4

Uma máscara de sub-rede é um número de 32-bit que mascara um endereço IP e divide o endereço IP em endereço de rede e endereço de host. A máscara de sub-rede é feita definindo bits de rede para All “1 ” s e definindo bits de host para All “0 ” s. Dentro de uma determinada rede, dois endereços de host são reservados para fins especiais e não podem ser atribuídos a hosts. O endereço “0 ” é atribuído a um endereço de rede e “255 ” é atribuído a um endereço de difusão, e eles não podem ser atribuídos a hosts. Os endereços IPv4 podem ser representados em qualquer notação de expressando um valor inteiro de 32 bits. Eles são mais frequentemente escritos no ponto-notação decimal, que consiste em quatro octetos do endereço expressa individualmente em decimal e números separados por períodos. Por exemplo, o endereço IP com quatro pontos 192.0.2.235 representa a versão decimal de 32 bits do número 3221226219, que no formato hexadecimal é 0xC00002EB. Ele também pode ser expresso em formato de hexadecimal com pontos, como 0xC0.0x00.0x02.0xEB, ou com valores de bytes como 0300.0000.0002.0353. Notação CIDR combina o endereço com o seu prefixo de roteamento em um formato compacto, no qual o endereço é seguido por um caractere de barra (/) e a contagem consecutiva de 1 bits do prefixo de roteamento (máscara de sub-rede).

Qual é o propósito do endereço IP de rede e do endereço de broadcast em uma rede IPv4?

endereço permite a comunicação com todos os hosts dessa rede. Para enviar dados a todos os hosts em uma rede, um host pode enviar um único pacote que é endereçado ao endereço de broadcast da rede. Além do endereço de broadcast 255.255.255.255, há um endereço IPv4 de broadcast para cada rede. Chamado de broadcast direcionado, esse endereço usa o endereço mais alto da rede, que é o endereço onde todos os bits do host são 1s. Por exemplo, o endereço de transmissão direcionado para 192.168.1.0/24 é 192.168.1.255.

Quais são os cinco principais tipos de classes de endereços IPv4 e como eles são usados?

Originalmente, o espaço do endereço IP foi dividido em poucas estruturas de tamanho fixo chamados de "classes de endereço". As três principais são a classe A, classe B e classe C. Examinando os primeiros bits de um endereço, o software do IP consegue determinar rapidamente qual a classe, e logo, a estrutura do endereço:

  • Classe A: Primeiro bit é 0 (zero);
  • Classe B: Primeiros dois bits são 10 (um, zero);
  • Classe C: Primeiros três bits são 110 (um, um, zero);
  • Classe D: (endereço multicast): Primeiros quatro bits são: 1110 (um, um, um, zero);
  • Classe E: (endereço especial reservado): Primeiros quatro bits são 1111 (um, um, um, um).

Qual é a diferença entre um endereço IPv4 público e um endereço IPv4 privado? Dê exemplos de cada um.

Endereços IP públicos são usados ao interagir com a internet, enquanto os endereços IPs privados operam em uma rede local. Tanto endereços IP públicos e privados permitem que os dispositivos se comuniquem entre si. Em uma rede comum, o roteador usa um endereço IP público para identificar você para o resto da internet, garantindo que e-mail, sites, conteúdo de streaming e outros dados cheguem até você corretamente.

O que é o NAT (Network Address Translation) e qual é a sua função em uma rede IPv4?

O NAT (Conversão de Endereços de Rede) é um modo de rede projetado para conservar o endereço IP mapeando uma porta e um endereço IP externo para um conjunto muito maior de endereços IP internos. Basicamente, um NAT usa uma tabela de fluxo para rotear o tráfego de um número da porta e endereço IP externos (host) para o endereço IP interno correto associado a um ponto de extremidade na rede (máquina virtual, computador, contêiner etc.)
