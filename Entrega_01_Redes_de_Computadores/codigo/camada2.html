Camada 02 (Enlace de Dados)

Bridge

Enlace de dados é feito a partir de um dispositivo de camada que permite e a ligação entre 02 segmentos

de rede, formando uma ponte(bridge) entre eles para que formem uma unica rede

Em redes antigas onde o meio utilizado era um hub "burro" o uso de bridges permitia uma maior divisão das redes

reduzindo assim o número de colisões e melhorando o desempenho da rede

o uso de bridges tem caido devido ao uso dos switchs que realizam a função criando segmentos individuais com menor

incidência de colisões

Switchs

Os switchs são dispositivos usados na segunda camada e possui caracteriísticas como:

  • Realiza a comutação.
  • Alivia Congestionamento.
  • Reduz tráfego de rede.
  • Elimina a necessidade de Hubs.
  • Utlizam infraestrutura de rede existente.
  • Conceta segmentos de rede LAN.
  • Usa endereço MAC para definir rede.
  • Os segmentos são livres de colisões./li>
  • Varios hosts pode se comunicar de forma simultãnea com o switch.
  • Encaminha quadros entre os hosts.
  • Possui entre 8 e 48 portas de conexão(podendo ser ainda maior de acordo com a necessidade.
  • Segmenta a rede interna.

Comutação

A comutação é um metodo de encaminhamento de dados eficiente e direcionado. Ao Invés de de todos

os dispositivos compartilharem o mesmo meio de comunicação , como no caso dos Hubs, a comutação permite

que sejam criadas conexoes diretas e dedicadas para transferência de dados.

Funcionamento do Switch

O switch usa o endereço MAC de cada host para fazer o endereçamento dos quadros.

No momento em que é ligado o switch , sua tabela de endereços está vazia e continuará assim até que o host transmita

informacões para um destino.

No inicio ele encaminha o quadro para todos os segmentos conectados, nesse momento já foi armazendao o endereço de origem

na tabela, o endereço de destino será armazenado logo em seguida.

Sendo assim, podemos afirmar que o switch aprendeo endereço de cada host lendo o endereço de origemno quadro transmitido para ele

e identificando a porta utilizada.

O Switch armazena o host e a hora em que ele foi armazenado em sua CAM, e o mantem lá por um período de tempo específico

se o endereço do host não é solicitado por um periodo de tempo ele é descartado da lista.

Gerenciamento da memória do Switch

Pode acontecer de mais de um quadro chegar para omesmo host;

O switch possui um buffer para direcionar os quadros para aporta correta e gerenciar se mais de um quadro chegar namesma porta.

O gerenciamento dessa memória pode ocorrer de 2 formas:

  • Buffer dedicado para cada porta.
  • Gerencia de memória

Buffer dedicado para cada porta

Cada porta tem seu buffer, mas se chegarem muitosquadro esse buffer pode “estourar” e a informaçãoque está na fila pode se perder

Gerenciamento de Memória

Cada porta tem seu buffer, mas também tem umbuffer global. Se o buffer de uma porta “estourar”, osquadros não se perdem; elas vão para o buffer global.

Modo de encaminhamento de quadros

O modo como os quadros são encaminhados peloswitch é chamado de switching.

Temos os seguintes tipos:

  • Store and forward
  • Fragment free
  • Cut through

Store and forward

Recebe bits do quadro no buffer e encaminha para o destino caso o quadro esteja ok.

Fragment Free

É similar ao cut, porem espera o recebimento dos primeiros 64 bytes ante sde encaminhar o quadro, caso não retorne problemas finaliza o envio do quadro

Cut Through

Faz o envio do quadro assim que chega ao switch, assim q o recebe ja lê o endereço MAC do destino e já o envia.
